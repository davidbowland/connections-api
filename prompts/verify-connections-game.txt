<checks>
Examine each category in the game and check for the following issues:

1. **Embedded substring categories** - If a category has non-empty `embeddedSubstrings`, every word must contain every listed substring. Also check whether the category name implies an embedded pattern (e.g. "contains MITT", "words with BLUE inside") but `embeddedSubstrings` is empty or missing — flag this as a problem.

2. **Pattern categories** (e.g. `[profession] + letter`, `[X] minus a letter`, `starts with a color`) - The varying element (the letter, color, etc.) must genuinely differ across words. Flag if all words use the same varying element (e.g. all profession plurals ending in S when the category claims different letters, or all words starting with the same color).

3. **Obviousness** - Flag if 2 or more words together would immediately betray the category to a casual solver without any lateral thinking required (e.g. "BAILIFF" and "JUDGE" in the same category).

4. **Category uniqueness** - Flag if two categories are semantically too similar to each other (e.g. one category for "dog breeds" and another for "types of dogs").

5. **Category alignment** - Flag if the category is something like "____ WORD" when the categoryConstraint is "Things with a shared property"

6. **Hint quality** - Each category's `hint` must be a creative question that evokes the category's key concept. Flag if a hint:
  - References, describes, paraphrases, or alludes to ANY of the 4 words in the category (e.g. mentioning "dancer" when FLAMENCO is a word, or "photographer" when PHOTOGRAPHER is a word)
  - Contains any word from the category name
  - Describes the category's MECHANIC or STRUCTURE rather than evoking its concept. This includes any hint that uses words like "follow", "precede", "come before", "come after", "share", "common", "connect", "complete", "make a phrase", "make a word", "make an expression", or asks what things "have in common"
  - Could apply interchangeably to any other category of the same type (e.g. a hint that works for ANY "word before ___" category is too generic — a good hint is specific to THIS category's connecting word)
  - Gives away the answer rather than nudging towards a key word, phrase, or idea

  A good hint points at the category concept without revealing the words:
  - "Do you see a color theme?" for "Black-and-white things" (evokes the visual concept)
  - "Is there an ending at the beginning?" for "BACK ___" (wordplay on the connecting word)
  - "Got a sugar craving?" for "Words after SWEET" (evokes the connecting word)
  - "What keeps your car shiny?" for "Wax ___" (evokes the connecting word)

  A bad hint describes the mechanic instead of evoking the answer:
  - "Could a single word follow each of these?" ← describes the pattern, not the concept
  - "What word precedes all four of these to make common expressions?" ← describes the pattern
  - "Is there a word that completes all four of these phrases?" ← describes the pattern
  - "Could any words share an adjective?" ← describes the pattern

  Hints should be about the contents of a category, not the style.
